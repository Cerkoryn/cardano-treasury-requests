# Cardano Treasury Requests

A small Vite app for tracking Cardano treasury withdrawal requests against the 350M ADA Net Change Limit.

The app has two views:

- Treasury Flow: a D3 Sankey-style flow from total requested ADA into individual proposals.
- Voting Priorities: a local, draggable priority table with per-proposal vote choices saved in browser localStorage.

## Data

Proposal data is generated by `fetch_cardano_budget_proposals.py` and written to:

```text
public/proposals-sankey.json
```

The script combines:

- Ekklesia Cardano Budget 2026 proposals
- Live on-chain treasury withdrawal proposals
- Ratified on-chain treasury withdrawal proposals scoped to the current NCL period

GitHub Actions runs the refresh workflow daily and commits `public/proposals-sankey.json` only when the data changes.

## Local Development

Install dependencies:

```bash
npm ci
```

Refresh proposal data:

```bash
npm run refresh:data
```

Run the app locally:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

## Deployment

The app is deployed as a static Vite site on Vercel.

Expected Vercel settings:

```text
Build Command: npm run build
Output Directory: dist
Install Command: npm ci
```
